# 修复规则

## 分类执行
- 零风险（删文件/改配置）：直接做
- 低风险（加权限/加校验）：改完 curl 验证
- 高风险（改业务逻辑）：必须先写测试再改代码

## 每个修复的流程
1. 说清楚改哪个文件、改什么、为什么
2. 改之前先读目标文件
3. 改完用 grep/ls 验证改动生效
4. 高风险改动：先写 tests/test_xxx.py，跑一遍确认红灯，再改代码，再跑确认绿灯

## 禁止事项
- 禁止一个 commit 混多个不相关修复
- 禁止改了 A 顺手"优化" B
- 禁止删代码不验证引用
- 禁止跳过验证直接说"完成"

## 高风险清单（必须有测试保护）
- PNL 计算
- JSON 解析
- 异常处理改造
- 交易配对逻辑
- 冷却/限流逻辑
